# Factory

## Q. 팩토리 패턴이란 무엇인가요?

### 답변

팩토리 패턴은 객체 생성 책임을 클라이언트 코드에서 분리해 추상화하는 디자인 패턴입니다.

클라이언트가 직접 구체 클래스를 생성하지 않고 인터페이스나 추상 타입에 의존하게 하여, 결합도를 낮추고 변경 범위를 줄일 수 있습니다.

특히 팩토리 메서드 패턴은 상위 클래스나 인터페이스에서 객체 생성 메서드의 틀을 정의하고, 하위 클래스에서 구체적으로 어떤 객체를 생성할지 결정하는 방식입니다.

### 키워드

- 추상화
- 인터페이스
- 구체 클래스 의존도 감소
- OCP
- 팩토리 메서드
- DI

## Q. 팩토리 패턴을 사용하면 어떤 장점이 있나요?

### 답변

팩토리 패턴은 객체 생성 책임을 분리해 클라이언트가 구체 클래스에 직접 의존하지 않게 하므로 결합도를 낮추고 변경 범위를 줄일 수 있습니다.

## Q. 팩토리 패턴과 전략 패턴의 차이는 무엇인가요?

### 답변

팩토리 패턴은 객체 생성 책임을 분리하는 패턴이고, 전략 패턴은 실행할 알고리즘이나 행위를 교체 가능하게 만드는 패턴입니다.

## Q. Spring에서는 팩토리 패턴을 어떻게 활용할 수 있나요?

### 답변

Spring에서는 인터페이스와 구현체를 Bean으로 등록하고 DI로 주입받아 사용하기 때문에, 직접 객체를 생성하지 않고 구체 클래스 의존도를 낮출 수 있습니다.
