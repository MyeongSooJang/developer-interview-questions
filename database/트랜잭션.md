# 트랜잭션

## Q. 트랜잭션이란 무엇인가요?

### 답변

트랜잭션은 데이터베이스의 상태를 변화시키는 하나의 논리적인 작업 단위입니다.

하나의 트랜잭션 안에는 여러 SQL 연산이 포함될 수 있고, 모든 연산이 성공하면 commit을 통해 결과가 반영됩니다.

반대로 중간에 하나라도 실패하면 rollback을 통해 이전 상태로 되돌리며, 이를 통해 데이터 정합성을 보장할 수 있습니다.

### 키워드

- 논리적 작업 단위
- commit
- rollback
- all or nothing
- 데이터 정합성
- ACID

## Q. 트랜잭션의 ACID란 무엇인가요?

### 답변

ACID는 트랜잭션이 보장해야 하는 원자성, 일관성, 격리성, 지속성을 의미합니다.

## Q. commit과 rollback은 무엇인가요?

### 답변

commit은 트랜잭션의 변경 내용을 데이터베이스에 확정하는 것이고, rollback은 트랜잭션 중 발생한 변경을 취소하고 이전 상태로 되돌리는 것입니다.

## Q. Spring에서 트랜잭션은 어떻게 사용할 수 있나요?

### 답변

Spring에서는 `@Transactional`을 사용해 메서드 단위로 트랜잭션 경계를 설정할 수 있습니다.
