# OSI 7계층

## Q. OSI 7계층이란 무엇인가요?

### 답변

OSI 7계층은 ISO에서 정의한 네트워크 통신 참조 모델입니다.

네트워크에서 데이터가 송신자에서 수신자까지 전달되는 과정을 7개의 계층으로 나누어 설명합니다. 각 계층은 자신이 맡은 역할에 집중하고, 위아래 계층과 정해진 인터페이스를 통해 데이터를 주고받습니다.

이렇게 계층을 나누면 네트워크 문제를 분석할 때 어느 계층에서 문제가 발생했는지 파악하기 쉽고, 각 계층의 프로토콜이나 장비를 독립적으로 이해하고 개발할 수 있습니다.

예를 들어 물리 계층은 케이블이나 전기 신호를 다루고, 데이터 링크 계층은 MAC 주소 기반의 프레임 전달을 담당합니다. 네트워크 계층은 IP 주소 기반 라우팅을 담당하고, 전송 계층은 TCP나 UDP를 통해 종단 간 데이터 전송을 담당합니다. 그 위의 세션, 표현, 응용 계층은 애플리케이션 간 통신, 데이터 표현, 사용자 서비스와 관련됩니다.

### 키워드

- ISO
- 네트워크 통신 참조 모델
- 7개의 계층
- 계층별 역할 분리
- 장애 원인 파악
- 프로토콜 독립성
- 캡슐화

## Q. OSI 7계층은 각각 어떤 역할을 하나요?

### 답변

OSI 7계층은 아래에서부터 물리 계층, 데이터 링크 계층, 네트워크 계층, 전송 계층, 세션 계층, 표현 계층, 응용 계층으로 구성됩니다.

| 계층 | 이름 | 역할 | 대표 예시 |
| --- | --- | --- | --- |
| 7계층 | 응용 계층 | 사용자가 접근하는 네트워크 서비스를 제공 | HTTP, HTTPS, FTP, SMTP, DNS |
| 6계층 | 표현 계층 | 데이터의 형식 변환, 압축, 암호화, 복호화 담당 | SSL/TLS, JPEG, MPEG |
| 5계층 | 세션 계층 | 애플리케이션 간 연결 세션을 생성, 유지, 종료 | Session, Socket |
| 4계층 | 전송 계층 | 종단 간 데이터 전송, 신뢰성, 흐름 제어 담당 | TCP, UDP |
| 3계층 | 네트워크 계층 | IP 주소 기반으로 목적지까지 경로를 선택 | IP, ICMP, Router |
| 2계층 | 데이터 링크 계층 | 같은 네트워크 내 인접 노드 간 프레임 전달 | Ethernet, MAC, Switch, Bridge |
| 1계층 | 물리 계층 | 데이터를 전기적, 물리적 신호로 전송 | Cable, Hub, Repeater |

면접에서는 모든 계층을 길게 설명하기보다, 하위 계층은 실제 전송에 가깝고 상위 계층은 애플리케이션 서비스에 가깝다고 설명한 뒤 TCP, IP, HTTP가 어느 계층에 속하는지 예시를 드는 것이 좋습니다.

## Q. 물리 계층은 무엇을 담당하나요?

### 답변

물리 계층은 데이터를 0과 1의 비트 단위로 전기적, 물리적 신호로 변환해 전송하는 계층입니다.

케이블, 커넥터, 허브, 리피터처럼 실제 물리 연결과 신호 전달에 관련된 장비가 이 계층에 해당합니다. 이 계층에는 IP 주소나 MAC 주소 같은 논리적 주소 개념은 없고, 신호를 어떻게 전달할지가 핵심입니다.

## Q. 데이터 링크 계층은 무엇을 담당하나요?

### 답변

데이터 링크 계층은 같은 네트워크 안에서 인접한 노드 간 데이터를 안정적으로 전달하는 계층입니다.

데이터를 프레임 단위로 다루고, MAC 주소를 이용해 목적지를 구분합니다. 대표적인 장비로는 스위치와 브릿지가 있고, 대표적인 프로토콜로는 Ethernet이 있습니다.

오류 검출도 이 계층에서 수행할 수 있지만, 종단 간 신뢰성을 보장하는 TCP와는 역할이 다릅니다.

## Q. 네트워크 계층은 무엇을 담당하나요?

### 답변

네트워크 계층은 IP 주소를 기반으로 목적지까지 데이터를 전달하는 계층입니다.

서로 다른 네트워크를 거쳐 목적지까지 패킷이 이동할 수 있도록 경로를 선택하는 라우팅을 수행합니다. 대표적인 프로토콜은 IP와 ICMP이고, 대표적인 장비는 라우터입니다.

즉, 네트워크 계층의 핵심은 목적지 IP까지 어떤 경로로 패킷을 보낼지 결정하는 것입니다.

## Q. 전송 계층은 무엇을 담당하나요?

### 답변

전송 계층은 송신 애플리케이션과 수신 애플리케이션 사이의 종단 간 데이터 전송을 담당합니다.

대표적인 프로토콜은 TCP와 UDP입니다. TCP는 연결 지향 방식으로 3-way handshake, ACK, 재전송, 흐름 제어, 혼잡 제어를 통해 신뢰성 있는 전송을 제공합니다. 반면 UDP는 비연결형 방식으로 신뢰성은 보장하지 않지만 구조가 단순하고 지연 시간이 짧아 실시간 통신에 자주 사용됩니다.

전송 계층에서는 포트 번호를 사용해 한 서버 안에서도 어떤 애플리케이션으로 데이터를 전달할지 구분합니다.

## Q. 세션 계층은 무엇을 담당하나요?

### 답변

세션 계층은 애플리케이션 간 통신 세션을 생성, 유지, 종료하는 역할을 담당합니다.

즉, 양쪽 애플리케이션이 대화를 시작하고 유지하며, 필요할 때 연결을 정리하는 논리적인 통신 관리를 담당한다고 볼 수 있습니다.

실제 TCP/IP 모델에서는 세션 계층의 역할이 응용 계층이나 전송 계층에 포함되어 구현되는 경우가 많습니다.

## Q. 표현 계층은 무엇을 담당하나요?

### 답변

표현 계층은 애플리케이션이 주고받는 데이터의 표현 방식을 처리합니다.

대표적으로 문자 인코딩, 데이터 형식 변환, 압축, 암호화와 복호화가 이 계층의 역할입니다. 예를 들어 JPEG, MPEG 같은 데이터 형식이나 SSL/TLS 같은 암호화 기술을 표현 계층의 예시로 설명할 수 있습니다.

## Q. 응용 계층은 무엇을 담당하나요?

### 답변

응용 계층은 사용자가 직접 사용하는 네트워크 서비스를 제공하는 계층입니다.

대표적으로 웹 통신에 사용하는 HTTP와 HTTPS, 파일 전송에 사용하는 FTP, 메일 전송에 사용하는 SMTP, 도메인 이름을 IP 주소로 변환하는 DNS 등이 있습니다.

사용자와 가장 가까운 계층이며, 애플리케이션이 네트워크를 사용할 수 있도록 서비스를 제공하는 역할을 합니다.

## Q. OSI 7계층과 TCP/IP 4계층의 차이는 무엇인가요?

### 답변

OSI 7계층은 네트워크 통신 과정을 설명하기 위한 개념적 참조 모델이고, TCP/IP 4계층은 실제 인터넷에서 많이 사용하는 구현 중심 모델입니다.

TCP/IP 4계층은 네트워크 인터페이스 계층, 인터넷 계층, 전송 계층, 응용 계층으로 구성됩니다.

OSI 모델의 물리 계층과 데이터 링크 계층은 TCP/IP의 네트워크 인터페이스 계층에 대응됩니다. OSI의 네트워크 계층은 TCP/IP의 인터넷 계층에 대응되고, OSI의 전송 계층은 TCP/IP의 전송 계층에 대응됩니다. OSI의 세션, 표현, 응용 계층은 TCP/IP의 응용 계층에 포함된다고 볼 수 있습니다.

| OSI 7계층 | TCP/IP 4계층 | 대표 예시 |
| --- | --- | --- |
| 응용, 표현, 세션 계층 | 응용 계층 | HTTP, HTTPS, FTP, DNS |
| 전송 계층 | 전송 계층 | TCP, UDP |
| 네트워크 계층 | 인터넷 계층 | IP, ICMP |
| 데이터 링크, 물리 계층 | 네트워크 인터페이스 계층 | Ethernet, MAC, Cable |

중요한 점은 HTTP나 HTTPS는 네트워크 계층이 아니라 응용 계층에 속하고, TCP와 UDP는 전송 계층, IP는 네트워크 계층 또는 TCP/IP 모델의 인터넷 계층에 속한다는 것입니다.

## Q. 데이터가 전송될 때 캡슐화는 어떻게 진행되나요?

### 답변

캡슐화는 상위 계층의 데이터에 하위 계층의 헤더를 붙여 전송 가능한 형태로 만드는 과정입니다.

예를 들어 사용자가 HTTP 요청을 보내면 응용 계층의 데이터에 전송 계층의 TCP 헤더가 붙어 세그먼트가 됩니다. 여기에 네트워크 계층의 IP 헤더가 붙으면 패킷이 되고, 데이터 링크 계층의 Ethernet 헤더와 트레일러가 붙으면 프레임이 됩니다. 마지막으로 물리 계층에서 비트 신호로 변환되어 전송됩니다.

수신 측에서는 반대로 각 계층의 헤더를 제거하면서 원래 데이터를 복원하는 역캡슐화가 진행됩니다.

## Q. 면접에서 OSI 7계층을 왜 알아야 하냐고 물으면 어떻게 답하나요?

### 답변

OSI 7계층은 실제 네트워크 장애를 계층별로 분리해서 분석하는 데 도움이 됩니다.

예를 들어 케이블이 빠졌다면 물리 계층 문제이고, 같은 네트워크 안에서 MAC 주소 기반 통신이 안 된다면 데이터 링크 계층 문제일 수 있습니다. IP 라우팅이 안 되면 네트워크 계층 문제이고, 포트 연결이나 TCP 연결이 실패하면 전송 계층 문제로 볼 수 있습니다. HTTP 응답 코드나 API 요청 문제가 발생하면 응용 계층 문제로 접근할 수 있습니다.

즉, OSI 7계층은 단순 암기 대상이라기보다 네트워크 문제를 구조적으로 분해해서 원인을 찾기 위한 기준입니다.
