# https://www.google.com 접속 시 일어나는 과정

## Q. https://www.google.com 접속 시 일어나는 과정은 무엇인가요?

### 답변

브라우저는 입력된 URL을 파싱해서 프로토콜, 도메인, 경로를 분리합니다.

도메인만으로는 서버와 통신할 수 없기 때문에 DNS Lookup을 통해 IP 주소를 조회합니다. 
이때 브라우저 캐시, OS 캐시, hosts 파일 등을 먼저 확인하고, 없으면 DNS 서버를 통해 IP를 찾습니다.

IP 주소를 얻으면 서버와 TCP 3-way handshake로 연결을 맺고, HTTPS이므로 TLS 핸드셰이크를 통해 서버 인증서를 검증하고 암호화에 사용할 키를 교환합니다.

연결이 완료되면 브라우저는 HTTP 요청을 보내고, 서버는 요청을 처리해 HTML, CSS, JavaScript 등의 응답을 반환합니다.

브라우저는 응답받은 HTML과 CSS를 파싱해 DOM과 CSSOM을 만들고, 렌더 트리 생성, 레이아웃, 페인트 과정을 거쳐 화면에 렌더링합니다.

### 키워드

- URL 파싱
- DNS Lookup
- 캐시
- TCP 3-way handshake
- TLS handshake
- 인증서
- HTTP 요청/응답
- DOM
- CSSOM
- 렌더 트리
- 레이아웃
- 페인트

## Q. DNS Lookup은 어떤 순서로 진행되나요?

### 답변

브라우저 캐시, OS 캐시, hosts 파일을 먼저 확인하고, 없으면 DNS 리졸버가 루트 네임서버, TLD 네임서버, 권한 네임서버 순으로 조회해 IP 주소를 찾습니다.

## Q. HTTPS에서 TLS 핸드셰이크는 왜 필요한가요?

### 답변

TLS 핸드셰이크는 서버 인증서를 검증하고 이후 통신을 암호화하는 데 사용할 키를 안전하게 교환하기 위해 필요합니다.

## Q. 브라우저 렌더링 과정은 어떻게 진행되나요?

### 답변

브라우저는 HTML과 CSS를 파싱해 DOM과 CSSOM을 만들고, 렌더 트리 생성, 레이아웃, 페인트 과정을 거쳐 화면에 표시합니다.
